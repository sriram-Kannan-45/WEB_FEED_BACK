1. OVERVIEW & TECH STACK
================================================================================
A role-based Learning Management System for Admins, Trainers, and Participants, 
focusing on Training Management, Feedback Collection, and Resource Sharing.

Frontend Stack:
- React.js 18
- Vite (bundler)
- React Router DOM (Routing)
- Chart.js (Admin analytics)
- CSS (Vanilla, custom scoped styling)

Backend Stack:
- Node.js & Express.js
- Sequelize ORM with MySQL
- JSON Web Token (JWT) + bcryptjs (Authentication)
- Multer (File uploads, using memory storage for DB encoding)

================================================================================
2. DETAILED ROLE-BASED WORKFLOWS
================================================================================

A. ADMIN WORKFLOW
-----------------
1. Login: Logs in using `admin@test.com` via `/api/auth/login`.
2. Dashboard Access: Directed to the comprehensive Admin Dashboard which includes:
   - Overview: Analytics charts (Average Trainer Rating, Participant Count)
   - Pending Approval: Approves new participant registrations.
   - Trainings: Creates, Edits, and Deletes training programs. Assigns specific trainers.
   - Participants: Views registered participants.
   - Feedback Reports: Aggregated view of all feedback submitted across all trainings.
   - Survey Config: Dynamically creates new feedback questions (Text, Rating, Multiple Choice) globally or mapped to specific trainings.
3. Trainer Creation: Admin creates Trainer accounts. The system auto-generates a username and a temporary password (displayed once).

B. TRAINER WORKFLOW
-------------------
1. Login: Logs in with auto-generated credentials provided by the Admin.
2. Dashboard Access: 
   - My Trainings: Views assigned trainings, checking capacity and dates.
   - Feedback Received: Reads participant feedback. Can submit a "Trainer Reply" to any feedback.
   - Upload Notes: Uploads study materials (file links or PDFs) mapped to a specific training.
   - My Profile: Edits personal details. Uploads a profile picture. (Images are encoded as base64 and saved directly into the database as LONGTEXT).
3. Profile Management: Updates phone, DOB, qualification, and experience.

C. PARTICIPANT WORKFLOW
-----------------------
1. Registration: Registers via the `/register` endpoint. Account goes into a `PENDING` state.
2. Approval: Must wait for the Admin to approve the account.
3. Login: After approval, logs in and accesses the Participant Dashboard.
4. Trainings: Views available trainings and clicks "Enroll".
5. Notes: Accesses approved notes/materials shared by Trainers.
6. Feedback: Submits post-training feedback (Trainer Rating, Subject Rating, Comments, and dynamically generated Survey Questions). Feedback can be submitted anonymously.

================================================================================
3. DATABASE SCHEMA & DATA FLOW
================================================================================

Users Table
- Stores all accounts. `role` ENUM (ADMIN, TRAINER, PARTICIPANT).
- `status` ENUM (PENDING, APPROVED) for access control.

TrainerProfiles Table
- One-to-One with User (Trainer).
- Stores DOB, Phone, Qualification, Experience.
- `image_path`: (LONGTEXT) Stores the Base64 representation of the Trainer's profile picture directly in the database.

Trainings Table
- Stores training details, start/end dates, capacity.
- Foreign Keys mapping to the Creator (Admin) and the Assigned Trainer.

Enrollments Table
- Maps Users (Participants) to Trainings.
- Status: ENROLLED or CANCELLED.

Feedbacks Table
- Maps Feedback to a Training, Trainer, and Participant.
- Contains quantitative (ratings) and qualitative (comments, trainer replies) data.

Notes / Resources Table
- Uploaded by Trainers, approved by Admins, viewed by Participants.

4. CRITICAL API ENDPOINTS & LOGIC
================================================================================

AUTHENTICATION
- POST /api/auth/register (Defaults to PARTICIPANT, PENDING)
- POST /api/auth/login (JWT Generation)

TRAINER PROFILE UPDATE (RECENTLY UPDATED)
- PUT /api/trainer/update
- Accepts `multipart/form-data`.
- Instead of saving to the filesystem, `multer.memoryStorage()` buffers the file. The buffer is converted to a `base64` string and stored directly in the `TrainerProfiles` table as a `LONGTEXT` column.
- Automatically handles empty string validation for `DATE` fields like DOB to prevent Sequelize `Invalid date` errors.

FEEDBACK & SURVEYS
- POST /api/survey (Admin creates dynamic questions)
- POST /api/feedback (Participant submits feedback)
- POST /api/feedback/:id/reply (Trainer responds to participant feedback)

ADMIN APPROVALS
- POST /api/admin/approve-participant/:id
- GET /api/admin/pending-participants

================================================================================
5. SECURITY PROTOCOLS
================================================================================

1. JWT Authorization: All protected routes require a Bearer token.
2. Role-Based Access Control (RBAC): Middleware verifies user `role` (ADMIN, TRAINER, PARTICIPANT) before executing controller logic.
3. Password Hashing: bcryptjs hashes passwords before DB insertion.
4. Input Sanitization: Empty strings for Dates are coalesced to `null` to prevent SQL/Sequelize validation crashes.
5. Storage Persistence: User-uploaded profile images are stored as Base64 in the Database to ensure persistence across cloud deployments without relying on local filesystem storage (`/uploads/` directory).
